1   What you’ll experience¶

  • Add new state to SEATSAT-managed robot, traverse to that state by voice operation and play the motion on Choreonoid

2   Related Tutorials¶

You’ll see very basic operation for SEAT (Speech Event Action Transfer) in this tutorial. See the following links for more details.

  • Format for the interactive state manager `SEAT`[1]

2.1   SystemEnvironment¶

  • Satisfy the environment defined in the previous tutorial

3   HowToRun¶

Your current directory is assumed to be the home of the USB unless specified.

For starting necessary services, quickly go [run-common-tools] and come back.

First, let’s run the robot status in the existing demo. Use the previsous demo again.

  1. Run “0_StartDemo2.bat” in ./demo/MotionByVoiceDemo (note that this isn’t “0_StartDemo.bat” we used previously).

  • ./demo/MotionByVoiceDemo/0_StartDemo2.bat |

  1. Run “1 ConnectRTC.bat”, “2 ActivateRTC.bat”, respectively

  • ./demo/MotionByVoiceDemo/1_ConnectRTC.bat
  • ./demo/MotionByVoiceDemo/2_ActivateRTC.bat

  1. Say “sai-sei yoro-shiku”. Unlike the previous tutorial, the robot bows down and put arms down. By the way it’s not visible on Choreonoid, but the state managed by SEAT is now dance_ready.

Command “sa-sei yoro-shiku” is valid from any robot state (eg. when only the right hand is up).

  1. Say “action yoro-shiku”. Robot starts the same dance from the previous tutorial, then goes back to the initial both_down state.

  1. Click “3_DeactivateRTC.bat”, “4_DisconnectRTC.bat”，”5_DemoExit.bat”, respectively.

  • ./demo/MotionByVoiceDemo/3_DeactivateRTC.bat |
  • ./demo/MotionByVoiceDemo/4_DisconnectRTC.bat |
  • ./demo/MotionByVoiceDemo/5_DemoExit.bat

4   Add new state¶

This portion represents the added state:

In ./demo/MotionByVoiceDemo/motion_cnoid2.seatml

  <state name="both_down">
    <rule>
  :
+    <rule>
+      <key>さいせい よろしく</key>
+      <command host="command">bow</command>
+      <statetransition>dance_ready</statetransition>
+    </rule>
  </state>
  :
+  <state name="dance_ready">
+    <rule>
+      <key>あくしょん よろしく</key>
+      <command host="command">SampleMotion1</command>
+      <statetransition>both_down</statetransition>
+    </rule>
+  </state>

Also, edit the config file in Julius to enable a new term action in the voice recognition:

In ./demo/MotionByVoiceDemo/motion2.grxml

+      <item>あくしょん</item>
:
+    </one-of>
+    <one-of>
:
+      <item>よろしく</item>
